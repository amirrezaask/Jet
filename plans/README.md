## Findings considered and rejected

- Replace React/CodeMirror with a native Rust renderer: rejected. It contradicts Jet's documented CodeMirror ownership model and would be a rewrite rather than a measured improvement.
- Add a GPU renderer now: rejected. Current bottlenecks are eager JavaScript, busy polling, lifecycle leaks, and DOM animation work; a renderer rewrite would not address those first-order costs.
- Remove Electron immediately after basic Tauri smoke passes: rejected. Native Tauri currently fails 6 shared E2E cases, excludes agents, has no Tauri CI, and lacks agent/extension parity.
- Micro-optimize ordinary Rust allocation before fixing lifecycle and polling: rejected. The measured and statically proven wins are higher-level: eliminate eager Shiki, 1 ms LSP polling, duplicate watchers, whole-DOM motion clones, and unbounded/repeated buffer scans.
